How to: Github + Hexo

1. 软件安装

使用的软件有:

  • Git
  • Node.js（ Should be at least Node.js 10.13, recommends 12.0 or higher ）

    具体安装过程省略，Node.js安装时一定要确保添加到环境变量：PATH 中。

    待上述的安装环境搭建好之后，接下来安装 hexo :

    $ npm install -g hexo-cli

也可以使用下面的命令安装：

$ npm install hexo

安装完成之后针对不同的系统有两种启动方式：

  • Windows and Mac:

      npx hexo <command>

  • Linux 设置 node_modules/的相对路径到环境变量中:

      echo 'PATH="$PATH:./node_modules/.bin"' >> ~/.profile

推荐安装最新版的 Hexo ，以及建议的 Node.js 的版本，如果非得用 Node.js 老版本，也不是不行，可以考虑安装 Hexo 的旧版本，具体适配情况如下：

2. github 部署

2.1. 实现原理

GitHub Pages 是由 GitHub 官方提供的一种免费的静态站点托管服务，让我们可以在 GitHub 仓库里托管和发布自己的静态网站页面。

而 Hexo 是一个快速、简洁且高效的静态博客框架，它基于 Node.js 运行，可以将我们撰写的 Markdown 文档解析渲染成静态的 HTML 网页。

2.2. SSH 密钥获取

首先创建 SSH 密钥，避免在博客部署时出现账号验证的情况出现，实现在终端就可以完成部署操作。

  • 创建 SSH :

      ssh-keygen -t rsa -C "GitHub 邮箱"

  • 添加 SSH :
    进入 [C:\Users\用户名.ssh] 目录（要勾选显示“隐藏的项目”），用记事本打开公钥 id_rsa.pub 文件并复制里面的内容。登陆 GitHub ，进入 Settings 页面，选择左边栏的 SSH and GPG keys，点击 New SSH key。

Title 随便取个名字，粘贴复制的 id_rsa.pub 内容到 Key 中，点击 Add SSH key 完成添加。

  • 使用 git bash here 验证 SSH 连接：

      ssh -T git@github.com

2.4. 初始化 Hexo

在安装完成 Hexo 之后，需要将 Hexo 初始化至目标文件中：
$ hexo init //初始化
$ cd //切换到指定的目录
$ npm install //安装组件

上述完成之后，使用下面的命令启动本地服务器进行预览：

hexo g   # generate 生成页面
hexo s   # 启动预览

在浏览器中输入 https://localhost:4000/ 即可到达博客页面：

Tips：如果出现页面加载不出来，可能是端口被占用了。Ctrl+C 关闭服务器，运行 hexo server -p 5000 更改端口号后重试。

2.5. 部署至Github Pages

通过上面的调试，本地博客可以成功使用，接下来就需要将其部署到 GitHub Pages 上以便通过网络访问。

  • 安装 hexo-deployer-git：

      npm install hexo-deployer-git --save

  • 修改 _config.yml 文件末尾的 Deployment 部分，修改成如下：

      # Deployment
  ## Docs: https://hexo.io/docs/one-command-deployment
  deploy:
      type: git
      repository: git@github.com:username/username.github.io.git
      branch: main

  • 完成后执行部署命令：

      hexo d  # deploy

完成！这时访问我们的 GitHub 域名 https://用户名.github.io 就可以看到 Hexo 网站了。
